Executive Summary

A comprehensive, production-ready API framework designed for rapid development, testing, and deployment of scalable web services. This project serves as a reference architecture for building high-concurrency APIs.

Multi-Database Support

Seamless switching between MongoDB, MySQL, and PostgreSQL via a unified abstraction layer.

Security First

JWT-based authentication with role-based access control (RBAC) and OWASP Top 10 protection.

Performance Ready

Containerized deployment with Go utilities for load testing and network validation.

Enterprise Monitoring

Built-in health checks, Prometheus metrics collection, and structured logging.

Quick Start Timeline

  • 5 min: Clone repository and install dependencies
  • 10 min: Configure environment and database connections
  • 2 min: Start services with Docker Compose
  • 1 min: Verify deployment with health check endpoints

MongoDB

Default NoSQL option. High flexibility for document storage.

Default Port: 27017

PostgreSQL

Robust SQL option for complex relational data and transactions.

Default Port: 5432

MySQL

Standard SQL option for traditional web applications.

Default Port: 3306

Deployment & Containerization

The project uses a multi-stage Docker build process to create a lightweight, production-ready image.

Multi-Stage Dockerfile
# Stage 1: Builder FROM node:20-alpine AS builder WORKDIR /app COPY package*.json ./ RUN npm ci --only=production # Stage 2: Go Utils FROM golang:1.21-alpine AS go-builder WORKDIR /go/src/app COPY go/ ./ RUN CGO_ENABLED=0 go build -o /go/bin/testutils ./cmd/testutils # Stage 3: Final Production Image FROM node:20-alpine WORKDIR /app COPY --from=builder /app/node_modules ./node_modules COPY --from=go-builder /go/bin/testutils ./go/bin/ COPY . . # Non-root user setup RUN addgroup -S nodejs && adduser -S nodejs -u 1001 USER nodejs CMD ["node", "index.js"]

Troubleshooting & Diagnostics

Common issues encountered during development and production deployment.

Symptom Possible Cause Solution
DB Connection Failures Network issues, wrong credentials, DB down Verify docker ps, check .env connection strings, test firewall.
High API Response Times Unoptimized queries, no caching, insufficient resources Check slow query logs, enable Redis caching, monitor CPU/Memory.
Auth Errors JWT config mismatch, token expiration Verify JWT_SECRET, check token expiration settings.
Container Crashes Memory limits, unhandled exceptions Check docker logs, verify resource limits, review error handling.
